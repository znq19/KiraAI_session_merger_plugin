# 会话合并与互通 · kira_session_merger（KSM）

[![Ask DeepWiki](https://deepwiki.com/badge.svg)](https://deepwiki.com/znq19/KiraAI_session_merger_plugin)

**版本 2.1.0** · 适用于 [KiraAI](https://github.com/) `core >= 2.6.1`

> 让 AI 在不同群聊、私聊里**真的是同一个人**——记得跨会话的经历，又能分清「现在在跟谁聊」。  
> 2.1 起：合并组内**逻辑一个窗口**（单一 Agent 队列），时间线按轮原子拼装，更接近官方单会话串行体验。

---

## 这是什么？

官方 KiraAI 默认：**每个群 / 每个私聊 = 独立短期上下文和记忆**。  
换一个窗口，她就像换了一个分身，不知道刚才在别处说过什么。

本插件在**不另建一套记忆库**的前提下，于每次请求模型前：

1. **读时合并**多个会话的官方短期上下文记忆 → 一条按时间交错的统一时间线（**统一按轮进行**，不拆开工具链）  
2. 告知模型**当前在哪个会话**（窗口锚点，防串台）  
3. 可选「偷看」未 @ 的闲聊（模拟真人刷手机之感）  
4. 把官方 `session_send` 改成 **跨会话路由（ROUTE）**：切到目标会话，带着合并上文继续干活  
5. **合并组单一 Agent 队列**：同一合并组同时只跑一个 agent（官方原版每个独立会话本也是如此），后来的会话排队，等当前会话**记忆落盘**后再处理  

**一句话：One Mind（同一人）+ 一条逻辑时间线 + 组内串行执行 + 读时合并。**

---

## 核心特点（为什么值得装）

| 特点 | 小白理解 |
|------|----------|
| **读时合并** | 不另存一份「总记忆」；各会话原记录还在，只是回答前拼起来看 |
| **同一人，不串台** | 记得别的群/私聊；本轮只在当前会话说话（锚点 + user 来源标签） |
| **组内单一 Agent 队列** | 像一个窗口：A 在跑时 B 的消息排队，A 落盘后再跑 B，减少叠 agent、重复调用工具等 |
| **时间线按轮原子** | 同一会话的 user→工具→回复整块排序，跨会话交错时不把工具链切断 |
| **assistant 不加会话前缀** | 对齐官方落盘习惯，减少模型学样输出 `[session:]` 污染 |
| **跨会话路由** | 「去私聊 / 去群里」→ 切到目标会话继续执行 |
| **交棒不双开** | 路由成功后源会话结束本轮工具循环 |
| **软重启 / 硬重开** | 上下文太长时可缩短；默认 soft 只砍「本次视图」 |
| **观察池偷看** | 未点名消息按概率注入，带来源标签 + `[msg_type: peek]` |
| **自动重开插件[ADS](https://github.com/znq19/KiraAI_auto_delete_session_plugin) 共存** | soft 默认不关 ADS；hard 自动关 ADS 防冲突（因为等于加强版的ADS Plus） |
| **历史查询工具** | 内置 `get_session_history`（OneBot HTTP） |
| **全部重启命令** | 默认 `/reboota`：清除所有参与合并的会话上下文（可配白名单与文案） |

## 推荐合并形态（四种玩法）

安装后在插件配置里调「合并范围」。会话 ID 格式：`qq:gm:群号` / `qq:dm:QQ号`。

### 1. 全部开启合并 —— 不再是分身，就是同一个人

| 配置项 | 建议 |
|--------|------|
| 启用会话合并 | 开 |
| 合并所有群聊 / 所有私聊 | 开 |
| 群聊与私聊互通 | 开 |
| 合并组单一 Agent 队列 | **开**（默认） |

### 2. 合并所有群聊 + 私聊 —— 随时了解各个群

| 配置项 | 建议 |
|--------|------|
| 合并所有群聊 / 私聊 | 开 |
| 群聊与私聊互通 | 开 |
| 最大合并会话数 | `6`～`12` |
| 其他会话超时 | 约 `20` 分钟 |

### 3. 合并所有私聊 + 特定群

| 配置项 | 建议 |
|--------|------|
| 合并所有群聊 | **关** |
| 合并所有私聊 | **开** |
| 群聊与私聊互通 | **开** |
| 额外会话 | 填重点群 `qq:gm:…` |

### 4. 只合并特定几个会话

| 配置项 | 建议 |
|--------|------|
| 合并所有群聊 / 私聊 | **关** |
| 额外会话 | **只填**要合并的 sid |
| 排除会话 | 可选 |

> **提示：**「群聊与私聊互通」只决定已参与合并的群与私是否同一时间线，**不会**单独把全部群/私勾上。范围 =「合并所有群/私」+「额外」−「排除」。

## 快速开始

1. 放到 `data/plugins/kira_session_merger/`  
2. 依赖（历史工具需要）：

```bash
pip install -r requirements.txt
```

3. 重启 / 重载插件  
4. WebUI → **会话合并** → **启用会话合并**  
5. 按上面形态配置范围；建议保持 **合并组单一 Agent 队列 = 开**  
6. 超限模式建议 **soft**（无 ADS 且想自动清上下文可用 hard）  
7. 可选：开启状态命令后发 `/merge s` 查看是否生效  

**历史工具：** 配置 OneBot HTTP 地址/端口/token。  
**全部重启：** 默认指令 `/reboota`（建议打开权限白名单）。

## 重要概念（小白版）

### 读时合并

- **不是**新建总记忆库  
- **是**回答前从官方各会话取出最近几轮 → 按时间拼成一条线 → 发给模型  
- soft 下各会话磁盘记录默认仍各自保存  

### 逻辑一个窗口（2.1）

同一合并组内：

```text
群A 在跑  →  群B 来消息  →  入队，不新开 agent
群A 记忆落盘  →  短暂 settle  →  再处理群B（上下文已含 A 刚写完的内容）
```

防抖 / 持续聊天仍按**每个会话**自己的 buffer（与 chat / sustained-chat 一致）；本插件只在「已经 flush 成 batch、即将开 agent」时做**组级串行**。

### 当前会话约束（锚点）

注入「你现在在哪个群/私聊」。  
带 `[msg_type: peek]` 的消息：偶然看到的其他会话内容，**不是当前会话发言和指令**，仅作背景感知。

### 来源标签

- **user**：合并时加 `[session: …]` / 群名或昵称等（与官方风格对齐）  
- **assistant / tool**：不加会话前缀（对齐官方落盘，减少学样污染）  
- **偷看**：同样来源格式 + `[msg_type: peek] [not_addressed_to_you]`  

### 跨会话路由（session_send）

1. 向目标投递跨会话请求（notice）  
2. 目标带着合并上文思考、调工具、说话  
3. 源会话本轮交棒结束（可简短确认也可不发）  
4. 短时间重复路由同一目标会被去重冷却拦截  

### 软重启 vs 硬重开

| 模式 | 做什么 | 风险 |
|------|--------|------|
| **soft** | 只缩短**本次**合并进模型的轮数 | 低，**但注意仅凭这样不会真正释放token** |
| **hard** | 合并组内相关会话删除并写回最近 N 轮 | **高**但能真正释放token，会自动禁用 ADS |

连续超限时保留轮数可自动减半（最少 1 轮）。  
合并构建超时可配置（默认 5 秒；hard 至少 8 秒）。

> **提示：**如果不想手动管理，请选择**hard**模式。

### 观察池 / 偷看

未 @ 消息进观察池；按概率注入。来源标签与合并时间线一致，并带 peek 标记。

## 配置一览（与 WebUI 对应）

### 合并范围

- 启用会话合并、允许的适配器  
- 合并所有群聊 / 所有私聊、群聊与私聊互通  
- 最大合并会话数、额外会话、排除会话  
- **跨会话请求去重冷却（秒）**

### 上下文与软硬重启

- 最大合并轮数、token 上限、超限保留轮数  
- soft / hard、检测间隔、字符/token 估算  
- 其他会话超时（及私聊/群聊专用）  
- 偷看概率、观察池、单次偷看条数  
- 工具轨迹、来源标签、跨会话时间交错  
- **合并构建超时（秒）**

### 合并组 Agent 队列（2.1）

- 合并组单一 Agent 队列（默认开）  
- 组锁超时（秒，默认 180）  
- 落盘后调度等待（秒，默认 0.4）  
- 组队列最大长度  

### 当前会话提示

- 是否注入锚点、会话说明文案（可自定义）  

### 历史查询工具

- 开关、HTTP、token、主人/白名单/限制群、缓存  

### 命令

- 状态 / 预览命令（默认均关闭）  
- **全部重启** `/reboota`：开关、关键词、权限、成功/拒绝/错误文案  
- 命令白名单（状态/预览）；全部重启有独立白名单  

### 调试 / 兼容

- 详细日志、合并预览日志、发送过程日志  
- 是否额外自动禁用 ADS（hard 时会强制关）

| `/merge s` / `/合并状态` | 合并组、超时、偷看等 | **关** |

## 常见问题

**Q：两个群同时 @，她怎么处理？**  
A：同一合并组内**排队串行**：先到的先跑，后到的等前一个会话**记忆落盘**后再跑，并带上完整合并线。

**Q：开启后 token 会不会暴涨？**  
A：会比单会话高。用最大合并会话数、超时、轮数/token 上限控制。

**Q：路由后源会话没说话？**  
A：允许。可简短确认也可不发；任务在目标会话继续。

**Q：和官方 session_send 有什么区别？**  
A：官方偏弱上下文发一句；合并模式是「切到目标 + 合并上文 + 继续执行」。

**Q：如何确认在工作？**  
A：日志可见 `[MERGER] built/applied`、`[MERGER queue] begin/deferred/release/replay`、`cross-session ROUTE`、`handoff stop` 等。

---

## 注意事项

1. 因对框架逻辑改动较大，默认总开关 **enabled = false**，需手动开启。  
2. 合并越多越费 token 与延迟。  
3. **hard 具有破坏性**；`/reboota` 会清空所有合并会话记忆，务必配权限。  
4. 历史工具依赖 OneBot HTTP。  
5. 令人放心的是：本插件不修改 KiraAI 本体与其它插件源码。
