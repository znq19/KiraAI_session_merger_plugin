# 会话合并 · kira_session_merger（KSM）
[![Ask DeepWiki](https://deepwiki.com/badge.svg)](https://deepwiki.com/znq19/KiraAI_session_merger_plugin)
**版本 2.0.0** · 适用于 [KiraAI](https://github.com/) `core >= 2.6.1`

> 让 AI 在不同群聊、私聊里**真的是同一个人**——记得跨会话的经历，又能分清「现在在跟谁聊」。

---

## 这是什么？

官方 KiraAI 默认：**每个群 / 每个私聊 = 独立短期上下文和记忆**。  
换一个窗口，她就像换了一个分身，不知道刚才在别处说过什么。

本插件在**不另建一套记忆库**的前提下，于每次请求模型前：

1. **读时合并**多个会话的官方短期上下文记忆 → 一条按时间交错的统一时间线  
2. 告知模型**当前在哪个会话**（防串台）  
3. 可选「偷看」未 @ 的闲聊（模拟真人刷手机之感）  
4. 把官方 `session_send` 改成 **跨会话路由（ROUTE）**：切到目标会话，带着合并上文继续干活（不是简单扔一句话）

**一句话：One Mind（同一人）+ 选择性在场 + 读时合并。**

---

## 核心特点（为什么值得装）

| 特点 | 小白理解 |
|------|----------|
| **读时合并** | 不另存一份「总记忆」；各会话原记录还在，只是回答前拼起来看 |
| **同一人，不串台** | 记得别的群/私聊刚刚发生过什么，但本轮只在当前会话说话 |
| **跨会话路由** | 用户说「去私聊 / 去群里」时，可无缝切到目标会话继续执行任务 |
| **交棒不双开** | 路由成功后源会话结束本轮工具循环，避免两边同时分身一样工作但她想先再说点什么就说 |
| **软重启 / 硬重开** | 上下文太长时可缩短；默认 soft 只砍「本次视图」，不乱删各会话本地记忆 |
| **观察池偷看** | 未点名消息可按概率带一点背景（带 `peek` 标记） |
| **[ADS（自动检查token重开会话插件）](https://github.com/znq19/KiraAI_auto_delete_session_plugin) 共存与强化** | 默认 soft 不关 ADS**更推荐下载**；仅 hard 模式会自动关 ADS 防冲突（因为它等于ADS Plus） |
| **历史查询工具** | 内置 `get_session_history`（OneBot HTTP），可替代重复的 history 插件 |

## 推荐合并形态（四种玩法）

安装后在插件配置里调「合并范围」。下面四种是最常用的组合（会话 ID 格式：`qq:gm:群号` / `qq:dm:QQ号`）。

### 1. 全部开启合并 —— 不再是分身，就是同一个人

**适合：** 就一个号一样、群不多、希望「人生一条线」。

| 配置项 | 建议 |
|--------|------|
| 启用会话合并 | 开 |
| 合并所有群聊 | 开 |
| 合并所有私聊 | 开 |
| 群聊与私聊互通 | 开 |
| 额外会话 / 排除会话 | 一般留空 |

效果：所有群 + 所有私聊共享一条时间线，跨会话记得住、能路由。

---

### 2. 合并所有群聊 + 自己的私聊 —— 随时了解各个群

**适合：** 多群活跃，私聊主要是主人/自己，希望群与私聊互通。

| 配置项 | 建议 |
|--------|------|
| 合并所有群聊 | 开 |
| 合并所有私聊 | 开（或只保留主人相关私聊，其余用「排除会话」） |
| 群聊与私聊互通 | 开 |
| 最大合并会话数 | 按群数量调，如 `6`～`12`（越大越全、越费 token） |
| 其他会话超时 | 建议 `20` 分钟左右，冷群自动不进上下文 |

效果：在私聊里也能接上「刚才某群发生了啥」；在群里也能记得私聊交代过的事。

---

### 3. 合并所有私聊 + 特定群 —— 链接你和这个群的群友们

**适合：** 私聊是主战场，只深度绑定一两个重点群。

| 配置项 | 建议 |
|--------|------|
| 合并所有群聊 | **关** |
| 合并所有私聊 | **开** |
| 群聊与私聊互通 | **开**（私聊与下面「额外」的群同一条线） |
| 额外会话 | 填重点群，例如 `qq:gm:123456789` |

效果：所有私聊 + 指定群互通；其它群不参与合并（仍是独立会话）。

---

### 4. 只合并特定几个会话 —— 精准控制与分化

**适合：** 要严格分区（工作群 / 玩梗群 / 某几个好友），不要「全世界一条线」。

| 配置项 | 建议 |
|--------|------|
| 合并所有群聊 | **关** |
| 合并所有私聊 | **关** |
| 群聊与私聊互通 | 开或关均可（开 = 列表内群私同一线；关 = 仅同类型之间，一般开更直观） |
| 额外会话 | **只填**要合并的 `qq:gm:…` / `qq:dm:…` |
| 排除会话 | 可选，再挡掉误入的 |

效果：只有名单里的会话共享记忆；名单外完全独立，便于人设/权限分化。

> **提示：**「群聊与私聊互通」只决定**已参与合并的**群与私是否同一时间线，**不会**单独把「全部群/全部私」自动勾上。范围永远由「合并所有群/私」+「额外会话」−「排除会话」决定。

## 快速开始

1. 将本目录放到 KiraAI 的 `data/plugins/kira_session_merger/`（或按你的插件安装方式安装）  
2. 安装依赖（若环境未带 httpx）：

```bash
pip install -r requirements.txt
```

3. 重启 / 重载插件  
4. 在 WebUI 打开 **会话合并** → **启用会话合并**  
5. 按上面四种形态之一配置范围  
6. 建议先保持 **超限处理模式 = soft**  （但如果你没有ADS，我建议还是用Hard）
7. 群里或私聊发送 `/merge s` 或 `/合并状态` 查看是否生效  

**历史工具（可选）：** 若要用 `get_session_history`，请配置 OneBot HTTP 地址/端口/token，与 NapCat 等一致。

## 重要概念（小白版）

### 读时合并

- **不是**把所有聊天存进一个新数据库当唯一真相  
- **是**回答前：从官方原版各会话记忆里取出最近几轮 → 按时间拼成一条线 → 发给模型  
- 各会话磁盘上的记录默认仍各自保存（**soft** 模式）

### 当前会话约束

每次请求会注入「你现在在哪个群/私聊」，减少把 A 群的话当成 B 群刚说的。

### 跨会话路由（session_send）

合并开启后，官方「跨会话发消息」工具被接管为：

1. 向目标会话投递跨会话请求（notice）  
2. 目标会话带着**合并后的上文**自己思考、调工具、说话  
3. 源会话本轮交棒结束（AI可自行选择说几句确认，也可以不说，随她的判断）  
4. 短时间重复路由同一目标会被去重冷却拦截  

### 软重启 vs 硬重开

| 模式 | 做什么 | 风险 |
|------|--------|------|
| **soft** | 只缩短**本次**合并进模型的轮数；各会话独立记录基本不动 | 低，但是如果你没有ADS又不想手动管理，还是用↓ |
| **hard** | 合并组内相关会话 **删除并写回** 最近 N 轮（类似 ADS 组级重开） | **高**，开启时会自动禁用 ADS |

连续多次超限时，保留轮数可能自动减半（最少 1 轮），避免死循环爆 token。

### 观察池 / 偷看

未 @、未唤醒的消息可进观察池；按概率注入，并带 `[msg_type: peek]`，表示「偶然看到，不是正式对话」。

## 配置一览（与 WebUI 对应）

### 合并范围

- 启用会话合并  
- 允许的适配器（默认 `qq`）  
- 合并所有群聊 / 所有私聊  
- 群聊与私聊互通  
- 最大合并会话数  
- 额外会话 / 排除会话  

### 上下文与软硬重启

- 最大合并轮数、合并历史 token 上限、超限保留轮数  
- 超限处理模式 soft / hard  
- 超限检测间隔、字符/token 估算  
- 其他会话超时（及私聊/群聊专用）  
- 偷看概率、观察池条数、单次偷看条数  
- 工具轨迹、来源标签、跨会话时间交错  
- 跨会话请求去重冷却（秒）  

### 其它

- 当前会话说明文案（可自定义）  
- 历史查询工具（HTTP / 权限 / 缓存）  
- 状态/预览命令与白名单  
- 调试日志  
- 兼容：是否额外自动禁用 ADS（hard 时会强制关）

## 命令

| 命令（默认） | 作用 |
|--------------|------|
| `/merge s` 或 `/合并状态` | 查看合并组、超时、偷看等状态（默认关闭，kira精神非必要不命令） |
| `/merge p` 或 `/合并预览` | 预览合并历史（默认关闭，调试用） |

可在配置里改关键词；可设命令白名单。

## 兼容与依赖

- **KiraAI core** `>= 2.6.1`  
- **Python 依赖：** `httpx>=0.27.0`（历史工具）  
- **ADS（auto_delete_session）：** soft 默认共存；hard 自动禁用  
- **history_plugin：** 可自动关闭，避免与内置 `get_session_history` 重复  
- **QQ 增强「正在输入」：** 跨会话交棒时，本插件会调用增强插件已有接口尽快结束输入状态（**不修改**增强插件源码）

## 常见问题

**Q：开启后 token 会不会暴涨？**  
A：会比单会话高。用「最大合并会话数」「其他会话超时」「最大合并轮数 / token 上限」控制。日常用 **soft**。

**Q：路由切换会话后源会话没说话？**  
A：允许。设计上「有必要时可简短确认，也可以不发」；任务在目标会话继续。

**Q：和官方 session_send 有什么区别？**  
A：官方偏「弱上下文发一句」；合并模式是「切到目标会话 + 合并上文 + 继续执行」。

**Q：如何确认插件在工作？**  
A：开启命令后发 `/merge s`；日志中可见 `[MERGER] built/applied`、`cross-session ROUTE`、`handoff stop` 等。

## 注意事项

1. 默认 **enabled = false**，需手动开启。  
2. 合并会话越多、轮数越多，费用与延迟越高。  
3. **hard 具有破坏性**，生产环境优先 soft。  
4. 历史工具依赖 OneBot HTTP 可用。  
5. 本插件不修改 KiraAI 本体与其它插件源码；跨会话与输入状态通过官方钩子/实例方法兼容。
